# 地图页 UI 设计说明

日期：2026-03-24
页面：`miniprogram/pages/map/index`
状态：实施中

## 目标

地图页需要同时满足三件事：

1. 用户一进入页面，立即知道当前聚焦的是哪家店。
2. 用户可以快速判断这家店值不值得去，并立即导航。
3. 用户上滑后能顺畅比较附近商家，再切换当前焦点店。

这意味着地图页不能只追求“好看”，而必须优先保证：

- 当前店信息清楚
- 主 CTA 稳定可见
- 展开态确实能看到更多附近商家
- 点击附近商家后，焦点店切换明确

## 设计原则

### 1. 收起态看当前店

收起态是一张完整决策卡，必须稳定展示：

- 店名
- 评分 badge
- 人均 / 距离 / 商圈
- 一行标签
- 一到两行地址
- 主按钮：`导航到这`
- 次操作：`从收藏移除`
- 弱提示：`上拉查看附近商家`

如果空间不够，优先截断地址，不允许裁掉主按钮。

### 2. 展开态看附近列表

展开态顶部卡片切为紧凑模式，仅保留：

- 店名
- 评分
- 人均 / 距离
- 一行标签
- 一行地址

展开态不显示：

- 导航按钮
- 从收藏移除
- 额外说明文案

展开态的首屏目标是看见 3-5 条附近商家。

### 3. 点击附近商家后切回完整卡片

用户点击附近商家列表中的任意一条后：

- 该店成为新的当前店
- 地图切到该店 marker
- 底部面板切回收起态完整卡片
- 下次展开时，附近列表按新的当前店重新计算

## 布局约束

### 顶部工具区

- 左右边距：`24rpx`
- 城市胶囊高度：`76rpx`
- 搜索框高度：`72rpx`
- 工具区 gap：`12rpx`

### 统计条

- 数字强调，但权重低于当前商家卡片
- `查看清单` 为弱入口，不能与主 CTA 同级

### 地图区

- 作为背景承载层
- marker 和 callout 是前景信息

### 底部面板

#### tabBar 避让

项目使用系统 tabBar，因此底部面板不能再用“整块上移”的方式避让。

设计要求：

- 面板视觉上仍然贴底
- 面板容器保持 `bottom: 0`
- 通过面板内部底部留白避让系统 tabBar
- 主内容不得落在 tabBar 背后
- 主按钮不能被 tabBar 遮挡

#### 高度建议

- 收起态：`492rpx`
- 展开态：`840rpx`

收起态需要包含 tabBar 内部避让空间，因此不能只按卡片本身的视觉高度计算。

## 组件规则

### 店名

- 收起态：`34rpx / 700`
- 展开态：`28rpx / 700`
- 收起态最多 2 行
- 展开态优先 1 行

### 评分 badge

- 统一文案：`★ 4.4`
- 收起态：`56-62rpx`
- 展开态：`48-52rpx`
- 颜色分级统一使用暖橙体系

### 标签

- 高度：`44rpx`
- 字号：`22rpx`
- 最多 1 行

### 地址

- 收起态最多 2 行
- 展开态最多 1 行
- 超出必须省略

### 主按钮

- 高度：`88rpx`
- 字号：`30rpx`
- 必须始终完整可见

## 字体与对齐规范

### 1. 标题与评分 badge 对齐

- 店名与评分 badge 处于同一标题组
- 评分 badge 以标题首行顶部为参照，不使用垂直居中对齐
- 店名允许换行，但评分 badge 不允许被挤压或下沉
- 标题组优先保证：店名第一视觉，评分第二视觉

### 2. 元信息对齐

- `人均`、`距离`、`商圈` 属于同一层级
- 元信息行使用同一基线，不做上下错位排布
- `距离` 胶囊只做轻强调，不应比 `人均` 更重

### 3. 标签与地址对齐

- 标签单独成行
- 地址单独成行
- 标签与地址之间保持稳定的垂直节奏
- 长地址优先省略，不允许顶坏按钮区

### 4. 按钮文字居中

- 主按钮、次按钮、危险按钮都必须采用真正的容器居中
- 不依赖单纯 `line-height = height` 做视觉居中
- 文字必须在视觉中心，而非只在数学中心

### 5. 附近商家列表对齐

- 每条列表项使用顶部对齐，而非垂直居中
- 左侧店名与评分、距离形成清晰两行
- 右侧标签作为弱信息，不参与垂直居中竞争

### 6. 全局字体层级

- 页面标题：`44rpx / 700`
- 当前店标题：`34rpx / 700`
- 紧凑卡标题：`28rpx / 700`
- 正文：`24rpx`
- 元信息 / 标签：`22rpx`
- CTA：`30rpx / 600`

禁止事项：

- 同一块里出现 3 个以上接近的大字号
- badge 字号过大导致与标题抢焦点
- 弱信息和 CTA 处于同一视觉权重

## 验收标准

### P0

- 收起态主按钮完整显示
- `从收藏移除` 完整可见
- 面板不被 tabBar 遮挡
- 点击附近商家后能切换当前焦点店

### P1

- 展开态首屏可见 3-5 条附近商家
- 展开态顶部卡片明显更紧凑，但不丑
- 地址与长店名不会顶坏布局

### P2

- 评分、按钮、标签、地址之间的层级稳定
- 暖色强调集中于评分和主 CTA
- 页面不会因长文本而破版
